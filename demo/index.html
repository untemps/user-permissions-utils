user-permissions-utils

API Support
check permission

Current state of every permission the library exposes, read upfront via checkPermission and kept live through the Permissions API change event. To reset a permission to 'prompt', click the lock or info icon in the browser address bar → reset permissions.

Permission getters

Each button watches the permission with get…Permission({ signal, timeout }) and fires the matching browser API to surface the prompt. Whichever settles first wins — the change event, or the API result re-read with checkPermission — so the result always shows the real outcome (granted, denied, dismissed, or a friendly error) instead of timing out. abort cancels the pending watch; push has no in-page trigger.
